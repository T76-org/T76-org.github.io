BUS:CC:CAPture:DATA?​

Type: Query

Syntax:

BUS:CC:CAPture:DATA?

Accepted forms​

FormSyntax
Long formBUS:CC:CAPture:DATA?
Short formBUS:CC:CAP:DATA?

Query the CC bus for the next available captured message. If no messages are available, returns an error. Messages are returned as arbitrary data blocks with the following format:

  • Start timestamp (8 bytes, uint64_t): The timestamp of when message capture started, in microseconds.
  • End timestamp (8 bytes, uint64_t): The timestamp of when message capture ended, in microseconds.
  • Message decoding result (4 bytes, uint32_t): The result of decoding the message. Possible values are:
    • 0: Success
    • 1: Invalid K-code
    • 2: CRC error
    • 3: Timeout error
    • 4: Incomplete message
  • SOP (4 bytes): The Start of Packet identifier.
  • Pulse count (4 bytes, uint32_t): The number of pulses in the captured message.
  • Pulse widths (N * 2 bytes, uint16_t): An array of pulse widths in PIO cycles.
  • Data length (4 bytes, uint32_t): The length of the decoded data in bytes.
  • Decoded data (M * 1 bytes, uint8_t): The decoded message

Parameters​

No parameters.

Response​

Returns query-specific data described above.

CodeMessageMeaning
